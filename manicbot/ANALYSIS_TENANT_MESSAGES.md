## Результаты анализа

### 1. Кнопка «Панель админа» (adm_back)

| Файл | Строка | Контекст | В тенантном боте? |
|------|--------|----------|-------------------|
| message.js | 71 | handleAIChat: кнопка под ответом AI для стаффа | Да — уже заменено на back_m |
| message.js | 495 | admin_cancel_reason: кнопка «Назад» после отмены | Да — админ/мастер |
| message.js | 1045 | confirm all requests: кнопка под подтверждением | Да — админ/мастер |
| ai.js | 211 | executeAIAction: после ADM_CONFIRM_ALL для system_admin | Да — текст adm_back |
| ai.js | 233 | executeAIAction: после ADM_CONFIRM_ALL для admin | Да — текст adm_back |
| callback.js | 476, 642 | Админ-панель (внутренние экраны) | Только при открытой панели |
| ui/admin.js | много | Экран админки | Только внутри панели |

Итог: в сообщениях, приходящих в общий чат (AI, подтверждения), в тенантном боте не должно быть подписи «Панель админа» — везде использовать «Главное меню» (back_m).

### 2. Сообщение «нет доступа»

| Место | Поведение после предыдущего фикса |
|-------|-----------------------------------|
| callback.js noAccessMsg() | В тенанте: showWelcome (без текста). В главном: короткий sysadm_no_access |
| message.js /add_technical_support | В тенанте: showWelcome. В главном: sysadm_no_access |

Итог: в тенантных ботах сообщение о доступе не показывается.

### 3. i18n

- Текущий sysadm_no_access: «У вас нет доступа к этому разделу.» (без Cloudflare/ADMIN_CHAT_ID).
- Старый длинный текст в коде не используется (только в archive/ и доке).

## План тестов

1. **Юнит-тесты**: `npm run test` — все 142 теста.
2. **Ручная проверка (чеклист)**:
   - В тенантном боте: ответ AI → под сообщением кнопка «Главное меню», не «Панель админа».
   - В тенантном боте: нажатие на старую кнопку платформы → переход в главное меню, без сообщения «нет доступа».
   - В главном боте: при отсутствии прав — короткое «У вас нет доступа к этому разделу.»

## Фиксы для применения

1. **ai.js**: в ответах после ADM_CONFIRM_ALL использовать back_m вместо adm_back (строки 211, 233).
2. **message.js**: в потоках cancel reason и confirm-all использовать back_m вместо adm_back (495, 1045), чтобы под сообщениями в чате не было подписи «Панель админа».
