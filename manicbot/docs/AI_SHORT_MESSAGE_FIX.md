# Анализ: «AI временно недоступен» при коротких сообщениях

## Проблема

При отправке сообщений из 1 символа («f», «g») стафф видел: «🤖 AI временно недоступен. Используй кнопки панели.»

## Причина

- `runWorkersAI(ctx, userMessage, ...)` возвращает `null`, если `userMessage.length < 2` (в ai.js).
- В `handleAIChat` при пустом `aiText` для стаффа всегда показывалось одно и то же сообщение про недоступность AI, без различения «сообщение не отправляли в AI» и «AI реально не ответил».

## Исправление

1. В `handleAIChat`: для сообщений короче 2 символов не вызываем `runWorkersAI`, `aiReply = null`.
2. Для стаффа при пустом ответе:
   - если сообщение было коротким (`txtTooShort`) → показываем `ai_short_use_panel`: «Слишком короткое сообщение. Используй кнопки меню.»;
   - иначе → `ai_unavailable_use_panel`: «AI временно недоступен. Используй кнопки панели.»

Ключи i18n: `ai_short_use_panel`, `ai_unavailable_use_panel` (ru, ua, en, pl).
