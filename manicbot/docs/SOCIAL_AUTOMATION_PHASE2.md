# @manicbot_com Social Automation — Phase 2 (migration 0127)

Extends the existing Instagram autopilot to: post to **Instagram + Facebook**
(Telegram-approved), and **autonomously reply to IG/FB comments** — all LLM text
generated by `claude -p` on the ThinkPad (Max subscription, no API key).

## Architecture

```
ThinkPad cron (claude -p)                Cloudflare Worker (Meta engine)              Meta
─────────────────────────                ───────────────────────────────             ────
social-content-builder ── POST ─────────► /admin/messaging/social-draft  (seed slot)
comment-responder ◄─ GET ─────────────── /admin/messaging/comments-pending
                  ── POST ─────────────► /admin/messaging/comment-reply  (draft/escalate/skip)
tg-bot approve  ◄─ GET ───────────────── /admin/messaging/social-pending
               ── POST ────────────────► /admin/messaging/social-approve (approve/skip)
                                          autopilot (cron 15m): gen image → approval → publish IG+FB
                                          phaseSocialCommentReply (cron 15m): post drafted replies
                                          /webhook/ig  ◄── comment events (changes[] comments/feed)
```

**Single webhook owner:** Meta delivers all events (DMs + comments) to one
callback (`/webhook/ig`), owned by the Worker, which also holds the encrypted
tokens, Graph client, dedup, and approval state. The ThinkPad never holds Meta
tokens; it only talks to the Worker seams.

## Data model (migration 0127)

- **`social_comment_inbox`** — one row per inbound comment. `comment_id` unique
  (Meta id) doubles as the replay guard. status: `new → drafted → replied`
  (or `escalated` / `skipped` / `failed`).
- **`marketing_content_plan`** new columns: `approved_at` (Telegram approval),
  `fb_post_id` / `fb_permalink` (Facebook fan-out result).

## Worker phases (src/marketing/)

- `phaseInstagramAutopilot` (autopilot.js): `pending → [awaiting_approval] →
  ready → publishing → posted`. When `MARKETING_REQUIRE_APPROVAL=1`, a generated
  post parks in `awaiting_approval` and pings the owner; publish (IG container +
  FB photo) only after approval. `getFbCredentials()` resolves the FB Page token.
- `phaseSocialCommentReply` (social-comments.js): posts ThinkPad-drafted replies.
  Guardrails: kill-switch `SOCIAL_COMMENTS_AUTOREPLY_ENABLED`, ≤30 replies/hour,
  status-guarded UPDATE (no double-post), failures → `failed`.

## Channel modules (src/channels/)

- `facebook-publish.js` — `createPhotoPost` (`/{page}/photos`), `getFbPostPermalink`.
- `comment-ingest.js` — parse IG `comments` / FB `feed` change → `social_comment_inbox`.
- `comment-reply.js` — `replyToIgComment` (`/replies`), `replyToFbComment` (`/comments`).

## Seams (src/http/messagingHttp.js, Bearer MESSAGING_TOKEN | ADMIN_KEY)

| Method | Path | Purpose |
|---|---|---|
| GET  | /admin/messaging/comments-pending | inbox rows status=new |
| POST | /admin/messaging/comment-reply | {comment_id, action: draft\|escalate\|skip, reply_text?, classification?} |
| POST | /admin/messaging/social-draft | upsert a content-plan slot with a generated caption |
| GET  | /admin/messaging/social-pending | posts awaiting approval |
| POST | /admin/messaging/social-approve | {id, decision: approve\|skip} |

## Webhook subscription

IG-direct subscribe adds `comments`; FB Page subscribe adds `feed`; the daily IG
resubscribe (`IG_RESUBSCRIBE_FIELDS`) carries `comments`.

## OAuth scopes (Meta App Review required)

- Instagram (already requested): `instagram_business_content_publish`,
  `instagram_business_manage_comments`.
- Facebook (added): `pages_manage_posts`, `pages_manage_engagement`,
  `pages_read_engagement`.

## Env flags (wrangler.toml [vars] / secrets)

| Var | Default | Meaning |
|---|---|---|
| `MARKETING_REQUIRE_APPROVAL` | `1` | Telegram-approve posts before publish |
| `SOCIAL_COMMENTS_AUTOREPLY_ENABLED` | `0` | autonomous comment replies (kill-switch) |
| `MARKETING_FB_TENANT_ID` | unset | FB Page channel tenant (EAA token) |
| `MARKETING_FB_PAGE_ID` / `MARKETING_FB_ACCESS_TOKEN` | unset (secret) | manual FB override |
| `ANTHROPIC_API_KEY` | unset in prod | caption fallback is Workers AI; primary text is ThinkPad `claude -p` |

## Security notes (for SECURITY_FINDINGS at merge)

- **Autonomous public comment replies** are a Meta-policy risk surface →
  mandatory guardrails: kill-switch (default OFF), hourly rate limit, ThinkPad
  classifier escalates negative/complaint/legal/medical (no auto-reply, owner
  pinged), status-guarded dedup. The Worker only posts replies the ThinkPad
  explicitly drafted (`status='drafted'`).
- **New public webhook fields** (`comments`/`feed`): same signature verification
  as DMs (X-Hub-Signature-256), `claimMetaMessage` dedup, own-account comments
  skipped. Comment ingestion never executes anything — it only stores rows.
- Platform-automation queries on `social_comment_inbox` / `marketing_content_plan`
  carry `// tenant-scan-ignore` annotations: rows are processed per-row by their
  own `tenant_id`, tokens resolved per-row — no cross-tenant read.
